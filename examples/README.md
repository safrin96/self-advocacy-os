# Examples

Browse these to see what good input files look like and what the project produces.

Each scenario includes filled-in inputs, evidence, and a sample output.

## Scenarios

### negotiation/

A mid-career analyst preparing for a compensation and title negotiation. She absorbed a senior analyst's responsibilities when the previous person left. No title change, no pay adjustment. Six months of proof.

**Files:**
- `wins.md` - accomplishments and feedback
- `context.md` - decision-maker profile and workplace dynamics
- `ask.md` - primary ask, fallback, and walk-away line
- `project-notes.md` - detailed project documentation (evidence)
- `feedback.md` - quotes from managers, peers, stakeholders (evidence)
- `metrics.md` - measurable outcomes (evidence)
- `sample-output.md` - the negotiation prep packet Claude Code would generate

### promotion/

A PM II building a case for Senior PM. Cross-team program ownership, interim leadership coverage, and process tools adopted department-wide.

**Files:**
- `wins.md` - accomplishments and feedback
- `context.md` - manager profile, promotion cycle timing, level expectations
- `ask.md` - promotion ask, fallback, and non-negotiables
- `sample-output.md` - promotion case with evidence table, validators, proof gaps, 30-day visibility plan

### opportunity/

An analyst choosing between three competing options: a stretch role application, an unpaid panel, and a paid workshop.

**Files:**
- `opportunities.md` - three options with details and constraints
- `sample-output.md` - scored comparison, recommendation, and boundary language for declining

## Using the examples

Read through a scenario that matches your situation. Then fill in your own files in `inputs/` and `evidence/` using the same level of specificity.

The sample outputs show you what to expect. Your actual output will be tailored to your evidence and situation.
