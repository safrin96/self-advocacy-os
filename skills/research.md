# Skill: Research

## Purpose

Find what is known, what is missing, and what should not be claimed yet.

## What to read

Before building a packet, read:
- `inputs/`
- `evidence/`
- Relevant prior files in `outputs/`
- Relevant example files only if the user asks for a sample or the input files are still empty

Do not use example data as if it belongs to the user.

## Evidence categories

Classify evidence as:

- Strong: specific metric, written feedback, documented scope, saved artifact, named stakeholder, date
- Moderate: project note, verbal feedback recorded by user, indirect evidence, unsourced but specific claim
- Weak: vague effort, emotion, comparison to coworkers, unsourced general claim
- Missing: claim has no supporting file or no validator

## Research questions

For every major claim, identify:
- What happened?
- Who benefited?
- How do we know?
- Who can validate it?
- What changed because of it?
- What proof would make this safer to say?

## Guardrail

If evidence is missing, flag it. Do not smooth over the gap.
