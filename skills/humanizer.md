# Skill: Humanizer

## Purpose

Turn analysis into language the user can actually say.

## Rewrite rules

Use:
- Short sentences
- Specific nouns
- Direct asks
- One idea per sentence
- Calm boundaries
- Evidence before conclusion

Avoid:
- Overexplaining
- Defensiveness
- Therapy-speak
- Corporate filler
- Over-polished scripts
- Inspirational language

## Meeting-language pattern

Use this structure for asks:

1. Name the conversation
2. State the evidence
3. Make the ask
4. Offer a fallback if needed
5. Ask for the next step

Example:

"I want to discuss aligning my title and compensation with the scope I am carrying. Since January, I have owned reporting across two departments, increased my report ownership from 5 to 12 recurring reports, and supported three department leads directly. I would like to move into the Senior Analyst title and have compensation reviewed before August budget decisions. If that cannot happen this cycle, I want a written September review date and documented senior-level responsibilities."

## Boundary-language pattern

Use this structure for declining opportunities:

1. Acknowledge the invitation
2. State the standard
3. Name what would make it viable
4. Leave the door open only if useful

Example:

"Thank you for thinking of me. I am focusing this quarter on paid sessions with defined audiences and specific topics. If this becomes a compensated session with a clear audience and scope, I would be open to revisiting."

## Final check

The final language should sound prepared, not performed.
