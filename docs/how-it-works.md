# How This Project Works

## The idea

Most self-advocacy advice tells you to "speak up" or "be confident." That assumes the hardest part is opening your mouth.

Often the harder problem happens earlier: your evidence is scattered, you have never connected your work to business value, and you have never practiced the version of the conversation where the answer is no.

This project treats self-advocacy as a systems problem, not a personality problem.

## What Claude Code does differently from a regular chatbot

A regular Claude chat is one prompt, one answer. You paste your background, get a response, and start over next time.

A Claude Code project keeps everything connected:

- **Standing instructions** (CLAUDE.md) tell Claude Code how to behave across every conversation: never invent accomplishments, flag weak proof, cite sources, save outputs.
- **Input files** hold your situation, evidence, and ask. You fill them in once. Claude Code reads them every time.
- **Evidence files** are your supporting documentation. Claude Code reads across all of them to find patterns.
- **Outputs** are saved as files you can revisit, revise, and reuse.

The result: you build a case over time instead of starting from scratch every conversation.

## The three workflows

### 1. Negotiation

You provide your wins, the decision-maker's style, and your ask. Claude Code maps each win to its source and business value, flags missing proof, drafts a primary and fallback ask, and lists likely objections with prepared responses.

### 2. Promotion

Same evidence, different analysis. Claude Code detects which accomplishments show next-level patterns (leadership, ownership, cross-functional influence, process design), identifies who can validate each claim, and creates a 30-day visibility plan.

### 3. Opportunity decision

You list 2-4 options. Claude Code scores each on alignment, proof, relationship access, upside, time cost, and risk of unpaid extraction. It recommends the strongest option and drafts boundary language for declining the others.

### Role-play practice

After any workflow, you can ask Claude Code to role-play the decision-maker. It pushes back with realistic objections and scores your responses on clarity, proof, tone, and whether you ended with a clear next step.

## Why this matters

Women are often told to be more confident in rooms where they have never been given a practice round.

Practice changes the nervous system. The first time you hear the objection should not be the moment your compensation, title, or opportunity is being decided.

This project gives you a private room to prepare.
